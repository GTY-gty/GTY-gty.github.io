hexo+github搭建博客步骤

发表于 2019-08-20

hexo+github搭建个人博客步骤

工具下载

nodejs下载

node.js官网下载，并配置环境变量
版本信息查看node -v、npm -v

Git下载

下载之前，先在GitHub官网注册GitHub账号（友情提示：需要翻墙）
我的账号：GTY-gty
Git官网下载，并配置环境变量。（友情提示：又要翻墙）
版本信息查看git --version

如果要用hexo等框架制作github pages的话，就要下载git，才能使用hexo的命令
如果不用框架，只要下载github客户端，即git的桌面版
操作只需点点点，非常方便~可以在github上发表博文，发布github pages

要想了解什么是Git，请点击这里

hexo安装

命令行工具：
可以用git自带的git bush（任意位置鼠标右击），或cmd（shift+鼠标右击）
用git bush不用配置git环境变量，用cmd就要配置
用npm下载hexo命令：
npm install hexo-cli -g
查看版本：hexo -v

本地生成hexo文件夹

在合适的文件夹下新建并初始化文件夹：
hexo init "文件夹名"（容易失败，可以试试npm cache clean --force清除缓存）
cd 到新建的文件夹下
安装必备组件：npm install
可以在_config.yml中修改一些配置信息

本地调试命令：hexo s，本地访问地址：localhost:4000

发布到github

安装hexo-deployer-git命令：
npm install hexo-deployer-git --save
修改配置文件：

发布到github pages命令，即部署到git服务器：
hexo clean清除缓存
hexo deploy发布

hexo主题下载：

NexT官网下载，并修改配置文件，详见NexT官网
新建博文命令：hexo new "博文名"
会在source/_posts下生成.md文件，直接编辑.md文件就可以了

参考资料

hexo官网：https://hexo.io
若hexo安装失败，参考资料：https://www.jianshu.com/p/6eba158ffeba（SSH之类的似乎并不需要）

vue安装步骤

发表于 2019-08-20

记录一下我一个月前vue的安装步骤~

当初的我还对这些乱七八糟的东西一窍不通，当然…现在也是。

下载nodejs

从官网下载nodejs,一路next安装
配置环境变量：path添加nodejs安装路径
node -v和npm -v查看版本

安装webpack

命令行全局安装webpack
npm install webpack -g

安装vue脚手架

版本2：
命令行全局安装vue cli2
npm install -g vue-cli
配置环境变量，安装完后cmd会显示安装路径
vue -V或vue --version查看版本
（若要安装版本3：npm install -g @vue/cli）
至此，vue的安装已经结束。

创建vue新项目

选择一个文件夹，在当前文件夹下打开cmd
初始化项目命令：
vue init webpack "项目名"
（vue cli3命令：vue create "项目名"，vue cli3是否不需要安装webpack？）
一路回车，有Y/n的选Y

安装UI框架

安装element-ui，每个项目安装一次，此框架是基于vue cli2的版本。
文件路径下npm install element -ui -S
VS Code打开package.json文件可以查看vue cli和element ui版本信息

setInterval()的写法

setInterval()方法至少要传递两个参数，第二个参数比较简单，是周期性调用第一个参数的时间间隔，然而第一个参数的写法却有多种。我们来做一个多选题，选项有点多，还请慢慢看：

以下哪些写法是正确的，可以使setInterval()方法成功执行

选项：A

选项：B

选项：C

选项：D

选项：E

选项：F

选项：G

选项：H

公布答案：除了D、G其他都是正确的。

原因分析

选项A是内置写法，正确是必然的。
选项BCDE考虑的是函数声明提前的问题。
选项FGH，抱歉我也不明白，setInterval()方法的第一个参数既可以传递函数（如选项ABCDE），也可以传递字符串（如选项FHG）。若选项ABCDE外套一个函数（如选项G），输出将是正确的，然而选项G却会报错：myFunction未定义。解决方法是：选项H。

函数的提前声明

1、使用函数声明的形式创建的函数function myFunction(){}，会被声明提前。也就是说，整个函数会在所有的代码执行之前就被创建完成，所以我们可以在函数声明之前，调用函数。
2、使用函数表达式创建的函数var myFunction = function(){}，不会被声明提前，所以不能在声明前调用。因为此时myFunction被声明了，且为undefined，并没有把function(){}赋值给myFunction。
